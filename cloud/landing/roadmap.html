Foundation — Core Protocol
SHIPPED · v2.15.1

The core identity bridge: portable passports, credential translation, policy engine, audit trail, OIDC federation, and webhooks. Deployed and running in production.

Agent Passport (URN scheme)
Portable identity credential valid across MCP, A2A, ANP, AG-UI. 3 tiers: permanent, session, ephemeral.
urn:aib:agent:{org}:{agent} · 9 API endpoints · Supabase Edge Functions
Credential Translation
Bidirectional translation between A2A Agent Cards, MCP Server Cards, and AG-UI Descriptors in under 1ms.
6 translation paths · _aib_source metadata · sub-millisecond latency
Ed25519 Audit Trail
Every action produces a cryptographically signed receipt. Persistent signing keys stored in PostgreSQL. SHA-256 hash chain.
RFC 8032 · PKCS8 persistent keys · tamper-evident chain
Policy Engine (12 rule types)
Declarative rules evaluated before every action. Deliverable gates, separation of duties, rate limits, time restrictions, domain control.
block / warn / log severity · denial receipts signed
OIDC Federation
Accept tokens from Google, Microsoft Entra, Okta, Auth0. JWKS verification with audience claim check.
RS256/384/512 · JWKS caching 1h · federation_trust table
Webhooks
4 event types with HMAC-SHA256 signed payloads: passport.created, revoked, policy.violation, translate.completed.
W3C DID:web Resolution
Every passport resolvable as a W3C DID v1.1 Document via did:web method. Ed25519VerificationKey2020 with publicKeyMultibase.
did:web:aib-tech.fr:agents:{slug} · /agents/{slug}/did.json · application/did+json
CI/CD Pipeline
GitHub Actions: 1,044 unit tests, security scan, 10 integration tests, auto-deploy to Netlify (dev → prod workflow).
Framework Integrations
Native tools for LangChain, CrewAI, and OpenAI Agents SDK. 6 tools per integration.
get_langchain_tools() · get_crewai_tools() · get_openai_agents_tools()

Decentralized Identity
NEXT UP

Make Agent Passports fully self-sovereign with W3C Verifiable Credentials, did:key support, and offline verification. No database required to verify an identity.

did:key Support
Self-contained DID method that encodes the public key directly in the identifier. No network resolution needed — ideal for ephemeral agents.
did:key:z6Mk... · Ed25519 multicodec · offline resolvable · SDK + Edge Function
W3C Verifiable Credentials
Each passport emitted as a W3C VC signed by the issuer. Verifiable offline without contacting the AIB server. Follows VC Data Model 2.0.
JSON-LD + Ed25519Signature2020 · credentialSubject: capabilities, protocols, bindings
VC Verification Endpoint
Dedicated endpoint to verify any VC — pure cryptographic check, no database lookup. Accepts VC JSON, returns valid/invalid + decoded payload.
StatusList2021 Revocation
Publish a compressed bitstring indicating which credentials are revoked. Enables revocation checks without real-time API calls.
W3C StatusList2021 · static endpoint · efficient batch revocation

Delegation & Trust Chains
PLANNED

Hierarchical delegation with Verifiable Presentations. An agent delegates to a sub-agent with narrowed permissions — the entire chain is cryptographically verifiable.

Delegation Credentials
New VC type: AIBDelegationCredential. Encodes delegator DID, delegatee DID, authorized capabilities (subset of parent), and validity period.
Verifiable Presentations (VP)
A delegatee presents a VP containing its own VC + the delegation VC + the parent VC. Verifier validates the entire chain in one operation.
Scope Narrowing Enforcement
Policy engine prevents privilege escalation: each delegation must reduce or maintain capabilities, never add.
Cascade Revocation via VC
Revoking a parent VC automatically invalidates all downstream delegation VCs through StatusList propagation.

Contextual Security
PLANNED

Credentials bound to task context. Ephemeral identities that expire with their mission. One-use credentials that self-destruct after presentation.

Task-Context Binding
Ephemeral credentials include a task_context hash (task_id, action, target, dataset). If the context changes, the credential is rejected.
One-Use Credentials
Credentials that can be presented exactly once. After use, automatically added to the revocation list.
Strict Auto-Expiration
Ephemeral credentials expire to the second. No grace period. SDK refuses to use them after expiration.

Semantic Audit & Intent
PLANNED

Audit trails that capture not just what happened, but why. Intent fields, invocation chains, data access logs, and EU AI Act compliance fields — all signed Ed25519.

Intent Fields in Receipts
Structured intent, invocation_chain (who invoked who), data_accessed (datasets touched), and risk_level — all signed in the receipt.
EU AI Act Compliance (Article 12)
Fields for human_oversight, decision_rationale, and affected_persons. Native compliance with EU AI Act record-keeping requirements.
Intent Inference API
LLM-powered analysis of agent actions: infer intent, detect anomalies, flag risk. Policy integration: block actions that don't match authorized intents.
Compliance Reports
Batch analysis endpoint: generate compliance reports from audit trail data. Exportable PDF/JSON for auditors.
